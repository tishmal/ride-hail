ride-hail

Три микросервиса взаимодействуют через PostgreSQL и RabbitMQ для управления всем рабочим процессом заказа поездок:

Ride Service — организует полный цикл поездки и управляет взаимодействием с пассажирами.
Служба определения местоположения водителя — управляет операциями водителя, алгоритмами сопоставления и отслеживанием местоположения в режиме реального времени.
Служба администрирования — обеспечивает возможности мониторинга, аналитики и контроля системы.

собираем проект:
docker compose down -v && docker compose up --build

✅ Компилируется go build -o ride-hail .
✅ Запускает три микросервиса (ride, driver-location, admin)
✅ Работает через RabbitMQ (только нужные очереди)
✅ Работает с PostgreSQL (минимальные таблицы)
✅ Реагирует на HTTP-запрос /rides и /drivers/{id}/online
✅ Посылает и принимает базовые сообщения в RabbitMQ
✅ Пишет структурированные JSON-логи
✅ Завершается корректно по Ctrl+C

docker ps | grep 5432 - найти контейнер который держит заданный порт
docker stop "id или имя контейнера"
docker rm "id или имя контейнера"

config.yaml — это файл конфигурации в формате YAML, где задаются настройки приложения, базы данных, сервисов и других компонентов.
